Machine Learning for Civil Engineers:
Term Project Report
Evaluating Classical vs. Deep Learning Architectures for Time-Series Streamflow Forecasting

Focus Areas: Tabular vs. Sequential Learning, Hyperparameter Tuning, Interpretability, and Bias-Variance Tradeoffs in Extreme Events

1. Executive Summary

This study evaluates the generalization capabilities of machine learning algorithms in predicting daily streamflow. The project contrasts two hydrologically distinct datasets (a snow-driven vs. rain-driven catchment) to analyze how varying physical underlying distributions impact ML model performance. The overarching goal was to determine which algorithmic architectures—Classical ML (Auto-Regressive Exogenous) or Deep Learning (Recurrent Sequential)—best map specific time-series dynamics.

Key ML Finding

Model architecture must match the underlying data variance. Tree-based ensembles (Random Forest, XGBoost) generalized exceptionally well on smooth, highly autocorrelated datasets (Snow). However, all models exhibited severe bias (underprediction) on sparse, high-variance target events (Rain), exposing the limitations of daily temporal resolution and Mean Squared Error (MSE) loss functions on extreme outliers.

2. Methodology & Experimental Design

2.1 Time-Series Data Splitting (10 / 3 / 2)

Standard k-fold cross-validation causes data leakage in time-series tasks. To ensure unbiased evaluation, the 15-year dataset was strictly partitioned chronologically:

  • Training Set (1999–2008): Used to fit initial model weights and decision boundaries.
  • Validation Set (2009–2011): Used exclusively for hyperparameter tuning via Grid Search and serving as the benchmark for Deep Learning Early Stopping.
  • Test Set "Vault" (2012–2013): Locked away until the final script to provide true generalization metrics.

2.2 Feature Engineering & Preprocessing

A standard scaler (StandardScaler) was applied to stabilize gradient descent for Neural Networks and regularize classical models. To allow classical ML to capture temporal dependencies, an Auto-Regressive Exogenous (ARX) approach was used, engineering a 7-day trailing lag window for all features, flattening the temporal dimension into tabular columns.

2.3 Model Architectures & Tuning

  • Classical Ensembles: Random Forest (Bagging) and XGBoost (Boosting) were tuned for n_estimators and max_depth to prevent overfitting on the training set.
  • Deep Learning: Simple RNN and LSTM networks were utilized. To prevent over-parameterization on a relatively small tabular dataset, dropout regularization and Early Stopping (monitoring validation loss) were implemented during backpropagation.

3. Final Generalization Performance

The models were evaluated on the unseen test set using the Nash-Sutcliffe Efficiency (NSE) metric. In ML terms, NSE behaves similarly to the R² score, where 1.0 represents a perfect prediction and 0 indicates the model is no better than predicting the mean of the target variable.

Observation: The Tree-based models achieved near-perfect mapping on the Snow catchment but failed to generalize on the Rain catchment. Deep Learning (LSTM) managed to extract slightly more signal from the Rain catchment, though absolute performance remained poor.

4. Model Interpretability (White-Box Analysis)

To understand the Tree models' success on the Snow catchment, Gini importance metrics were extracted from the Random Forest model.

The algorithm autonomously identified dayl(s) (Daylight duration) as a primary feature. From a machine learning perspective, daylight provided a highly stable, periodic signal (seasonality) that allowed the decision trees to cleanly threshold the slow-moving target variable (snowmelt) without overfitting to noisy daily temperature fluctuations.

5. Error Analysis: The High-Variance Problem

To diagnose the failure on the Rain Catchment, an extreme-event error analysis was conducted, stratifying the test set into "Normal" vs. "Extreme" (Top 10%) target values.

5.1 Bias and Loss Function Limitations

The models suffer from extreme bias on high-variance events. Because flash floods (Rain) are sparse outliers in the dataset, and the models were trained using MSE loss, predicting massive spikes carries an enormous mathematical penalty if the model guesses the wrong day. Consequently, the algorithms "learned" the safest route: conservatively predicting the historical mean to minimize overall loss, resulting in total failure during extreme events.

6. Final Engineering Takeaways

  1. Algorithm vs. Data Resolution: No amount of hyperparameter tuning or deep learning complexity can overcome a target variable whose temporal resolution (daily averages) destroys the underlying variance (hourly storm intensity).
  2. Interpretability: Classical ensemble methods (Random Forest) remain superior for tabular datasets where interpretability and feature extraction are required.
